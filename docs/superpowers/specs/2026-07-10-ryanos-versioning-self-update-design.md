# RyanOS ŌĆö Versioning + Git-Native Self-Update ŌĆö Design

**Date:** 2026-07-10
**Status:** Approved (pending spec review)
**Repo:** `naquin316/damon-ade` (RyanOS), `apps/desktop`

## Problem

RyanOS is a personal Electron cockpit that Ryan develops continuously (via Claude
Code, committing to `main`). Today:

- There is **no way to tell which build is running** from inside the app.
- Versioning is stale: `package.json` = `0.1.0`; the one git tag `v0.1.0` points at an
  old commit (`8dd78d3`), not HEAD; there is **no GitHub Release**.
- GitHub **Actions are disabled** on the fork (`actions/permissions ŌåÆ enabled:false`),
  so there is no cloud build/release pipeline.
- The app is **unsigned / ad-hoc signed**, so classic macOS auto-update
  (electron-updater / Squirrel.Mac) cannot apply updates.
- Keeping the installed `/Applications/RyanOS.app` current with committed work is a
  manual rebuild-and-reinstall each time.

We want: proper versioning + tagging, a visible version in the app, and an in-app way
to pull the latest and self-update.

## Constraints (decisions that shape everything)

1. **Single machine, git-native.** The app runs only on Ryan's own Mac(s), where the
   repo `~/Code/damon-ade` is present. No Apple Developer signing, no notarization, no
   cloud CI required. "Update" = fetch ŌåÆ local rebuild ŌåÆ reinstall ŌåÆ relaunch.
2. **Consume updates only.** The in-app control pulls latest and self-updates. It does
   NOT push/release (that stays in Claude Code / terminal).
3. **Version = semver tag + embedded commit/date.** A human semver (bumped/tagged at
   milestones) plus the exact commit + build date embedded at build time.
4. **Detached updater script** executes the rebuild (survives the app quitting).

## Non-goals (YAGNI)

- No Developer ID signing / notarization.
- No re-enabling GitHub Actions; no GitHub Release objects.
- No electron-updater download flow (existing `auto-updater.ts` stays **dormant**, not
  removed ŌĆö removing it risks disturbing `menu.ts` wiring for no benefit).
- No multi-machine / other-user distribution.
- No periodic background polling (check on launch + on demand only).
- No auto-stash of uncommitted work (the updater refuses on a dirty tree instead).

## Architecture

Four units, each independently understandable and testable:

### Unit A ŌĆö Build-time version stamping
- New script `apps/desktop/scripts/gen-build-info.ts`, run **before** compile in the
  `predev`, `prebuild`, and `pretypecheck` chains in `apps/desktop/package.json` (so the
  file is always fresh before any consumer compiles it ŌĆö dev, build, and typecheck).
- It shells out to git and **overwrites** a **committed** module
  `apps/desktop/src/shared/build-info.generated.ts`:

  ```ts
  export const BUILD_INFO = {
    version: "0.2.0",        // apps/desktop/package.json version
    commit: "535fa20",       // git rev-parse --short HEAD
    commitFull: "535fa20ŌĆ”",  // git rev-parse HEAD (for compare math)
    branch: "main",          // git rev-parse --abbrev-ref HEAD
    buildDate: "2026-07-10",  // ISO date at build time
    tag: "v0.2.0",           // git describe --tags --exact-match || ""
  } as const;
  ```

- The file is **committed with safe dev defaults** (`commit: "dev"`, empty `tag`, etc.)
  so a fresh checkout and any static `import` always resolve; `gen-build-info` overwrites
  it with real values before every dev/build/typecheck. It is **not git-ignored** ŌĆö
  regeneration churn on this one small file is accepted in exchange for a static,
  always-resolvable import. Consumers import it directly:
  `import { BUILD_INFO } from "shared/build-info.generated"`.
  The generated file carries a `// machine-generated by gen-build-info.ts` header. We
  commit the regenerated values whenever we cut a versioned build; day-to-day the churn
  is a single small file and is left as-is.

**What it does:** produce the single source of truth for "what am I running."
**Interface:** the `BUILD_INFO` object.
**Depends on:** git, package.json.

### Unit B ŌĆö In-app version display
- New tRPC router `appInfo` (`apps/desktop/src/lib/trpc/routers/app-info/`) with `get`
  returning `BUILD_INFO` merged with the current self-update status.
- New renderer component `VersionBadge`
  (`.../TopBar/components/VersionBadge/`) mounted in `TopBar.tsx`'s right-side cluster,
  next to `OrganizationDropdown`.
  - Idle: `v0.2.0 ┬Ę 535fa20` (muted).
  - Behind: appends `Ōåæ N behind` in an accent color.
  - Checking / updating: a spinner + label.
- Click opens a small popover: full build details (version, commit, branch, date),
  **Check for updates**, and ŌĆö when behind ŌĆö **Update & relaunch** plus a "view N
  commits" link to the GitHub compare URL
  (`https://github.com/naquin316/damon-ade/compare/<installed>...main`).

**What it does:** show the running build; entry point to check/trigger updates.
**Interface:** `appInfo.get` + `selfUpdate.*` tRPC; renders in `TopBar`.
**Depends on:** Unit A (build info), Unit C (status/actions).

### Unit C ŌĆö Self-update controller (main process)
- New module `apps/desktop/src/main/lib/self-update.ts`, separate from the dormant
  `auto-updater.ts`.
- New status enum in `apps/desktop/src/shared/self-update.ts`:
  `idle ┬Ę checking ┬Ę behind ┬Ę updating ┬Ę error`, plus a `behindCount` and optional
  `error` message on the event.
- New tRPC router `selfUpdate`
  (`apps/desktop/src/lib/trpc/routers/self-update/`): `getStatus`, `check`, `update`,
  and a `subscribe` observable (trpc-electron requires the **observable** pattern, per
  `apps/desktop/AGENTS.md`).
- **Repo path resolution:** read `~/.ade/self-update.json`
  (`{ "repoPath": "~/Code/damon-ade" }`); default `~/Code/damon-ade`; expand `~`;
  validate it is a git repo with an `origin` remote before use.
- **Check logic:** `git -C <repo> fetch origin main`; compare
  `BUILD_INFO.commitFull` to `git rev-parse origin/main`; behind count via
  `git rev-list --count <installedCommit>..origin/main`. If installed commit is `dev`
  or unknown to the repo, treat as "unknown" (offer update, skip the count).
- **Cadence:** one check on app launch (after ready) + on demand from the badge. No
  interval polling.

**What it does:** know whether we're behind and orchestrate the update handoff.
**Interface:** the `selfUpdate` tRPC router + status subscription.
**Depends on:** Unit A, Unit D (spawns it), `~/.ade/self-update.json`.

### Unit D ŌĆö Detached updater script (the engine)
- New script `apps/desktop/scripts/self-update.sh`, referenced directly from the repo
  at `<repoPath>/apps/desktop/scripts/self-update.sh` (repo is guaranteed present).
- Invoked by Unit C **detached** (`spawn` with `detached: true`, `stdio: 'ignore'`,
  `unref()`), with args: repo path, target app path (`/Applications/RyanOS.app`), and
  the app PID to wait on. On click the app writes an intent marker, spawns the script,
  shows "Updating, will relaunchŌĆ”", then quits.
- Script steps (all logged to `~/.ade/update.log`, with a start/end marker):
  1. Wait for the app PID to exit (bounded timeout, then proceed).
  2. **Guards** (abort, leave app intact, write failure marker): repo must be on
     `main` with a **clean working tree**; toolchain present (`bun` on PATH).
  3. `git -C <repo> pull --ff-only origin main`.
  4. `bun install` **only if** the lockfile changed in the pull.
  5. Build chain in `apps/desktop`: `bun run clean:dev`, `bun run compile:app`,
     `CSC_IDENTITY_AUTO_DISCOVERY=false bun run package`.
  6. **Swap only on build success:** `rm -rf /Applications/RyanOS.app`, then
     `cp -R release/mac-arm64/RyanOS.app /Applications/RyanOS.app` (copies the freshly
     built app directly ŌĆö skips the DMG mount for speed; the DMG is still produced by
     `package` as an archive artifact).
  7. `xattr -dr com.apple.quarantine` (defensive) and `open -a /Applications/RyanOS.app`.
  8. On success: clear intent marker. On any failure before step 6's swap: the old
     `/Applications` app is untouched; write a failure marker so the relaunched (old)
     app can surface "Update failed ŌĆö see ~/.ade/update.log."
- Supports `--dry-run`: print the plan (guards + steps) without mutating anything.

**What it does:** perform the git pull + rebuild + atomic-ish install + relaunch out of
process.
**Interface:** CLI args (repo, app path, pid) + `--dry-run`; exit code + markers/log.
**Depends on:** git, bun, electron-builder, the repo, `/Applications`.

### Failure-state signalling (markers)
- `~/.ade/update.intent`   ŌĆö written before quit; cleared on success.
- `~/.ade/update.failed`   ŌĆö written on failure, contains a short reason; read on next
  launch by Unit C to show a one-time error, then deleted.
- `~/.ade/update.log`      ŌĆö full transcript of the last run.

## Versioning & tagging discipline (local, no CI)

- `apps/desktop/package.json` semver is the source of truth.
- **Milestone release** = bump version ŌåÆ commit ŌåÆ annotated tag `v<version>` ŌåÆ push
  tag. Helper `apps/desktop/scripts/tag-release.sh [patch|minor|major|<x.y.z>]` does the
  bump + commit + annotated tag + `git push origin <tag>` locally. No GitHub Actions,
  no Release object.
- **Reconcile the stale tag:** leave `v0.1.0` (points at `8dd78d3`) as history; **bump
  to `0.2.0`** and tag current HEAD as the first build under this scheme, as part of
  implementation.

## Data flow (update path)

```
VersionBadge (renderer)
  ŌåÆ selfUpdate.check (tRPC)              ŌåÆ self-update.ts: git fetch + compare
  ŌåÆ status: behind (N)                   ŌåÆ subscription ŌåÆ badge shows "Ōåæ N behind"
  ŌåÆ user clicks "Update & relaunch"
  ŌåÆ selfUpdate.update (tRPC)             ŌåÆ write ~/.ade/update.intent
                                         ŌåÆ spawn self-update.sh (detached), pass pid
                                         ŌåÆ app.quit()
self-update.sh (detached)
  wait pid ŌåÆ guards ŌåÆ pull ŌåÆ build ŌåÆ swap /Applications ŌåÆ open -a
  (log to ~/.ade/update.log; markers on success/failure)
new build launches ŌåÆ gen-build-info baked new commit ŌåÆ badge shows new sha
```

## Error handling

- **Dirty tree / not on main:** updater refuses, writes `update.failed` with the
  reason; old app relaunches and surfaces it. (Protects uncommitted work.)
- **Build failure:** `/Applications` app never touched; failure marker + log; old app
  relaunches.
- **Toolchain missing (`bun` not found):** refuse early with a clear message.
- **Repo path invalid / no origin:** `check` returns an error status; badge shows
  "update unavailable" with a tooltip pointing to `~/.ade/self-update.json`.
- **Network down on `fetch`:** treat as "couldn't check," stay idle (no error toast).
- **`build-info` says `commit: dev`:** offer update without a behind-count.

## Verification

- **Unit A:** run `gen-build-info.ts`; assert all fields populated and match
  `git rev-parse`.
- **Unit C:** create a synthetic commit on `origin/main`; confirm `check` flips status
  to `behind` with the right count; confirm clean state reports `idle`.
- **Unit D:** `self-update.sh --dry-run` prints the plan and mutates nothing; test the
  dirty-tree and missing-toolchain guards abort correctly.
- **End-to-end (the `verify` ethos):** commit + push a trivial visible change, click
  **Update & relaunch**, confirm the app relaunches and the badge shows the new commit
  sha.
